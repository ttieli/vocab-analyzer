## 🔴 高优先级澄清项（阻碍开发）

### Q1: CEFR词汇表的具体数据源选择

**问题描述**:
规格中提到了多个可选数据源（EVP、Oxford 3000、CEFR-J），但没有明确：
- 这些数据源是否可以免费获取？
- 是否需要注册账号或申请API密钥？
- 如果首选EVP不可用，具体的降级策略是什么？

**影响**:
- 阻碍P1任务（获取CEFR词汇表）的执行
- 可能影响整个项目时间规划

**建议行动**:
1. 首先尝试访问 English Vocabulary Profile 官网，确认访问方式
2. 如果需要付费或注册，立即切换到 Oxford 3000 (开源免费)
3. 明确记录最终选择的数据源和理由

**需要决策**:
- [ ] 是否愿意为官方EVP数据付费？(如果需要)
- [ ] 是否接受使用开源但非官方的词汇表？
- [ ] 是否需要合并多个数据源以达到5000词？

### Q2: 词组词典的"可分离"标注标准

**问题描述**:
规格要求词组标注"是否可分离"(separable)，但没有明确：
- "可分离"的定义和判断标准是什么？
- 是否所有词组都必须有此标注，还是只针对动词短语？
- 如果数据源没有提供此信息，如何补充？

**示例**:
- "look up" - 可分离：look the word up ✓
- "give up" - 可分离：give it up ✓
- "take part in" - 不可分离：不能说 take in part ✗

**影响**:
- 影响P2任务（词组词典收集）的数据质量标准
- 影响后续Story 3（词组识别）的实现难度

**建议行动**:
1. 明确可分离的语言学定义
2. 如果数据源缺少此标注，准备手动标注或使用规则推断
3. 优先标注常用500个词组即可，罕见词组可后补

**需要决策**:
- [ ] 是否接受部分词组缺少separable标注？
- [ ] 是否需要手动标注高频词组？
- [ ] 可分离标注的准确率要求是多少？(如>90%)

### Q3: 中英词典的释义详细程度

**问题描述**:
规格要求"简洁释义（1-3个主要含义）"，但没有明确：
- 如何判断哪些是"主要含义"？
- 如果一个词有10个含义，如何筛选最主要的3个？
- 是否需要包含例句？

**示例对比**:
```
简洁版：
book (n.) 书籍

标准版：
book (n.) 1.书籍 2.账簿 3.剧本

详细版：
book (n.) 1.书籍；图书 2.账簿；账本 3.剧本；台词集
book (v.) 1.预订 2.登记
```

**影响**:
- 影响P1任务（中英词典准备）的工作量
- 影响后续Story 4（中文释义集成）的用户体验

**建议行动**:
1. 优先使用ECDICT等现成词典的释义，不做二次加工
2. 如果有多个释义，保留前3个最常用的
3. 暂时不包含例句（例句从原文提取更有意义）

**需要决策**:
- [ ] 接受词典原有的释义格式和详细度？
- [ ] 是否需要人工审核和精简释义？
- [ ] 词性是否需要全部标注（如一词多性）？

### Q5: 数据格式的统一标准

**问题描述**:
规格要求"统一为CSV/JSON格式"，但没有明确：
- CSV和JSON如何选择？各用于什么场景？
- CSV的分隔符用逗号还是制表符？
- JSON的嵌套层级如何设计？

**示例**:
```csv
# CSV格式（适合表格数据）
word,level,word_type,definition_cn
book,A2,noun,书籍
```

```json
// JSON格式（适合复杂结构）
{
  "vocabulary": [
    {
      "word": "book",
      "level": "A2",
      "word_type": "noun",
      "definition_cn": "书籍"
    }
  ]
}
```

**影响**:
- 影响所有数据任务的实施细节
- 影响后续程序读取数据的代码实现

**建议行动**:
1. 简单表格数据用CSV（词汇表、词组）
2. 复杂嵌套数据用JSON（映射表、元数据）
3. CSV统一使用UTF-8编码，逗号分隔

**需要决策**:
- [ ] 确认CSV编码为UTF-8 with BOM还是无BOM？
- [ ] JSON是否需要格式化输出（便于阅读）？
- [ ] 是否需要提供样例数据文件？

### Q6: 数据量的最低标准 vs 理想标准

**问题描述**:
规格中多次提到"至少5000词"、"至少500词组"，但没有明确：
- 这是最低可接受标准还是目标？
- 如果只找到4500词，是否可以接受？
- 理想情况下希望达到多少词汇量？

**影响**:
- 影响数据收集的工作量评估
- 影响是否需要合并多个数据源

**建议行动**:
1. 明确"至少5000词"是硬性要求
2. 如果单一来源不足，合并多个开源词汇表
3. 理想目标可设为8000-10000词

**需要决策**:
- [ ] 5000词是底线还是目标？
- [ ] 是否接受略低于5000的情况？
- [ ] 理想的词汇表规模是多少？
